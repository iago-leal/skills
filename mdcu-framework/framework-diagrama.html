§ 01

O ciclo canônico.

Fluxo macro · skills em composição

O framework opera em um ciclo de quatro etapas encadeadas. project-init estabelece o contrato técnico uma única vez (ou em refresh estrutural). MDCU conduz cada sessão de trabalho através de seis fases efêmeras. RSOP/SOAP destila o raciocínio em prontuário permanente. commit-soap sela o ciclo no histórico do repositório. A ordem não é sugestão: é condição.

§ 02

MDCU — seis fases, uma sessão.

Zoom sobre o núcleo · fases efêmeras · anamnese em disco

Todas as fases escrevem no arquivo transitório _mdcu.md — o prontuário de rascunho da sessão. É defesa contra Lost in the Middle: o substrato em disco é mais confiável que a memória de conversa.

§ 03

_mdcu.md — anatomia do prontuário.

Artefato efêmero · substrato em disco · destilado no SOAP
# Sessão 2026-04-18 — autenticação OAuth Tentativas de Reenquadramento: 0/2 ## F1 Preparação - ARCHITECTURE.md lido: sim - RSOP #3, #7 ativos - viés: apego à solução JWT prévia ## F2 Escuta S: Demandas - corrigir timeout em login intermitente Queixas - "sistema parece lento na madrugada" Notas - SIFE: frustração alta; ideia: "culpa do redis" - possível demanda oculta: confiança no stack ## F3 Exploração O: - logs prod: p99 de 2.3s entre 02h–04h - redis: saturação de conexões sem alert - código: pool fixo em 10, sem autoscale - rastreio: 1.não 2.não 3.sim 4.sim 5.sim ## F4 Avaliação - hipótese: saturação pool + job batch 02h ## F5 Plano - alt. A: pool dinâmico + alert saturação - alt. B: desacoplar batch para worker pool - decidido: A (menor risco, reversível) ## F6 Execução # destilação aqui → /rsop soap → delete
Contador 0/2 Disjuntor humano. Reenquadramento em F6 incrementa; ao 2/2, exit protocol obrigatório.
Campo S: Escrito durante a escuta, não retroativamente. Demanda ≠ queixa, sub-slot explícito.
Campo O: Fatos observáveis. Fonte declarada. Checklist de rastreio sempre 5 linhas.
Destino F6 relê o arquivo inteiro antes de chamar /rsop soap. SOAP é destilação, não memória.
Ciclo de vida Deletado após commit-soap. O raciocínio que importa mora no SOAP permanente.
§ 04

RSOP — prontuário do software.

Inspirado em Weed · RCOP e-SUS PEC · separação ativos/passivos

O Registro de Software Orientado por Problemas é o índice longitudinal. A separação entre arquivo ativo (injetado no contexto do agente) e arquivo morto (consultado sob demanda) é economia de tokens com propósito: o que está fechado não merece ocupar atenção.

Estrutura em disco
rsop/ ├── dados_base.md # perfil mínimo ├── lista_problemas.md # ATIVOS → CLAUDE.md ├── passivos.md # ARQUIVO MORTO ├── seguranca.md # opcional · mdcu-seg └── soap/ ├── 2026-04-15_rate-limit.md ├── 2026-04-16_oauth.md └── 2026-04-18_saturacao.md
Componente vivo
lista_problemas.md — ativos
Apenas problemas abertos. Injetado no CLAUDE.md do projeto. Cada token ali consome janela de contexto do agente — entrada é barata, permanência não.
Arquivo morto
passivos.md — fechados
Problemas resolvidos. Consultado só em dois casos: suspeita de regressão ou pedido explícito do usuário. Fora disso, invisível ao agente por design.
Destilado por sessão
soap/YYYY-MM-DD_*.md
S · O · A · P · R. A e P são 1:1 por problema (máx. 5 palavras cada). Prosa livre desqualifica o registro.
Migração
/rsop revisar
Move problemas resolvidos: lista_problemas.md → passivos.md. Com data de fechamento e SOAP que resolveu. Regressão reabre no sentido inverso.
§ 05

Dois registros, uma narrativa.

Micro-commits (ruído técnico) · commit-soap (marco cognitivo)

O histórico do Git carrega duas camadas sobrepostas. Manter a distinção é o que dá valor ao framework.

Camada operacional
git commit
Micro-commits atômicos durante F6. Checkpoints, WIPs, salvamentos antes de tentativa arriscada. Não exigem SOAP.
wip: extrair cálculo pool
refactor: nome de função
fix: typo em env default
chore: bump dep patch
Quando usar: sempre que um estado merece preservação dentro do ciclo de trabalho.
Camada cognitiva
commit-soap
Selo de fechamento de sessão ou merge de feature final. Formato A+P derivado do SOAP.
A: #3 saturação pool redis em batch 02h
P: #3 pool dinâmico + alert saturação

Refs: rsop/soap/2026-04-18_saturacao.md
Uso exclusivo: fechamento da sessão MDCU completa. Nunca para WIP.
Retrospectiva longitudinal $ git log --grep="A:" # todos os marcos cognitivos do projeto
$ git log --grep="#3" # história longitudinal do problema #3
$ git log --invert-grep --grep="A:" # audita ruído operacional
§ 06

O disjuntor humano.

Circuit breaker em F6 · teto de 2 reenquadramentos · exigência de decisão humana

Reenquadramento é propriedade do sistema, não falha. Mas reenquadramento encadeado sem âncora externa é sintoma de anamnese insuficiente — e de loop de API consumindo recursos sem convergir. O disjuntor força o agente a parar e devolver a decisão ao humano.

0/2
Estado inicial
Sessão começa com contador zerado.
Campo Tentativas de Reenquadramento: 0/2 no topo de _mdcu.md.
1/2
Primeiro reenquadramento em F6
Incrementa. Prossegue com nova hipótese.
Agente registra motivo, muda de enquadramento, retoma execução. Ainda dentro da margem.
2/2
Segundo reenquadramento em F6
Incrementa. Executa por último.
Se este também falhar e um novo reenquadramento for identificado, o próximo passo é abortar — não uma terceira tentativa.
⊗
Gatilho do disjuntor
Execução automática abortada.
Exit protocol obrigatório. Sem nova tentativa, sem nova hipótese, sem nova ferramenta. Decisão é do humano.
EXIT PROTOCOL · FORMATO OBRIGATÓRIO
[DISJUNTOR 2/2 — EXECUÇÃO ABORTADA] Tentativas no ciclo: 1. [enquadramento 1] → falhou: [motivo] 2. [enquadramento 2] → falhou: [motivo] 3. [próximo reenquadramento identificado — não executado] Gap atual: [o que não se sabe / o que está ambíguo] Decisões possíveis (sua escolha, não minha): - [opção A] - [opção B] - [opção C, incluindo abortar sessão] Aguardando decisão. Não prosseguirei sozinho.
§ 07

Delegação ao mdcu-seg.

Rastreio no MDCU · aprofundamento aqui · analogia Wilson-Jungner

O MDCU faz rastreio (5 itens binários — detecta sintoma). Quando dispara, a avaliação aprofundada, contenção e vigilância longitudinal são da skill mdcu-seg. Exatamente como um teste de rastreio positivo encaminha para avaliação especializada — nunca conduz o manejo dentro da atenção primária.

Fase MDCU Condição Ação
F1 #[A] de segurança ativo no RSOP /mdcu-seg auditoria — contextualizar; avaliar se é incidente ativo
F3 Rastreio item 1 (dados sensíveis) ou item 2 (auth/autz) afirmativo /mdcu-seg threat-model — STRIDE sobre o problema
F5 Alternativa proposta falha no rastreio sem mitigação /mdcu-seg threat-model — sobre a alternativa antes da decisão compartilhada
F6 Sinal de incidente durante execução (log anômalo, vazamento, atividade suspeita) /mdcu-seg incidente — protocolo F0; MDCU suspenso, _mdcu.md preservado
Qualquer Menção a vazamento, breach, pentest, CVE crítico, LGPD delegar conforme contexto
§ 08

O isomorfismo clínico.

Mapeamento MFC ↔ Software · o framework não é metáfora, é transposição

O framework é a transposição disciplinada de métodos da Medicina de Família e Comunidade para o desenvolvimento de software. A analogia não é retórica — é estrutural: ambos operam em ciclos longitudinais, com usuário como coautor, e degeneram em ruído sem registro orientado por problema.

Dimensão Medicina de Família Engenharia de Software
Método central MCCP — Método Clínico Centrado na Pessoa MDCU — Método de Desenvolvimento Centrado no Usuário
Artefato permanente Prontuário orientado por problemas (Weed, 1968) RSOP — Registro de Software Orientado por Problemas
Registro da sessão SOAP — Subjetivo, Objetivo, Avaliação, Plano SOAP — idêntico; A e P 1:1 por problema
Dado primário Demanda e queixa do paciente — fonte de diagnóstico Demanda e queixa do usuário — fonte de especificação
Rastreio Wilson-Jungner — condições prevalentes, evitáveis, tratáveis Segurança — 5 itens binários em F1, F3, F5, F6
Contrato inicial Ficha de identificação e anamnese de admissão ARCHITECTURE.md — stack, convenções, guardrails, lock file
Determinismo Receita médica exata — dose, via, frequência, fabricante Lock file commitado — versão pinada, hash, auditável
Encaminhamento Atenção primária → especializada quando rastreio dispara MDCU → mdcu-seg quando sinal de risco surge
Urgência Protocolo de emergência suspende manejo eletivo F0 (incidente) suspende MDCU; _mdcu.md preservado
Reavaliação Reenquadramento é propriedade do método, não falha Reenquadramento livre — exceto em F6 (disjuntor 2/2)
Fundamentação teórica

Wilson, J. M. G. & Jungner, G. (1968). Principles and Practice of Screening for Disease. OMS. Critérios aplicados ao rastreio de segurança no MDCU.

e-SUS PEC. Modelo RCOP adotado no registro do SUS — formato enxuto e telegráfico do SOAP clínico.
